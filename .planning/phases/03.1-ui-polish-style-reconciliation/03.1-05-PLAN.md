- Background circle: stroke color `var(--muted)` at 20% opacity, strokeWidth 8, fill none
- Foreground arc: stroke color from `getMasteryColor(score)`, strokeWidth 8, fill none, with strokeLinecap="round"

**Category Card (category-card.tsx):**

Redesign the card layout while keeping the same `CategoryCardProps` interface and Link wrapper:

1. **Top border accent on hover:** Replace the current `style={{ borderLeft: ... }}` with a Tailwind approach. The card gets `border-t-2 border-transparent` by default, and on hover it transitions to the mastery color. Since the mastery color is dynamic (from getMasteryColor), use inline style for the hover border: apply `border-t-2` and set `style={{ borderTopColor: 'transparent' }}` on the Card, then use a `group-hover` approach. Actually, since the color is dynamic CSS var, the cleanest approach:
   - Card: `className="group transition-all bloom-hover cursor-pointer h-full border-t-2"` with `style={{ borderTopColor: 'transparent' }}`
   - Add an `onMouseEnter`/`onMouseLeave` approach is too complex. Instead: remove the left border entirely. Add `border-t-2 border-transparent` as classes, and use a CSS custom property + hover:
   - Set `style={{ '--card-accent': color } as React.CSSProperties}` on the Card
   - In the Card className, use `hover:border-t-[var(--card-accent)]` — but this doesn't work with Tailwind arbitrary values from CSS vars cleanly.
   - **Simplest approach:** Keep `border-t-2` and use inline style with a state. Actually, the simplest approach that works: add a colored top-border div INSIDE the card that is initially opacity-0 and becomes opacity-100 on group-hover:
   ```tsx
   <div className="absolute top-0 left-0 right-0 h-0.5 opacity-0 group-hover:opacity-100 transition-opacity" style={{ backgroundColor: color }} />
   ```
   The Card needs `relative overflow-visible` (or use the existing overflow-hidden and make the border div inside).

