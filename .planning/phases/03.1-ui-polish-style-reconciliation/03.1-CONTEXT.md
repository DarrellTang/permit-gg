# Phase 3.1: UI Polish & Style Reconciliation - Context

**Gathered:** 2026-03-26
**Status:** Ready for planning

<domain>
## Phase Boundary

Reconcile the app interior's visual language with the landing page's cyberpunk/Minority Report aesthetic. No new features — purely visual consistency. The landing page is the design reference; the app interior moves toward it. This phase touches: base Card component, quiz components, dashboard components, summary components, answer options, and the app layout background.

</domain>

<decisions>
## Implementation Decisions

### Card & Surface Styling
- **D-01:** Update base Card component (card.tsx) to use `glass` + `ghost-border` by default — all cards get the treatment automatically
- **D-02:** Replace `surface-container` var backgrounds with glass effect (backdrop-blur + semi-transparent bg)
- **D-03:** Cards should feel like floating HUD panels, consistent with landing page's `bg-card/80 backdrop-blur-2xl` approach

### Glow & Interaction Effects
- **D-04:** Apply `bloom-hover` (pink glow on hover) to all interactive elements — buttons, cards, links
- **D-05:** No persistent glows inside the app — keeps usable for extended study sessions without visual fatigue (cyberpunk accessibility concern)
- **D-06:** Answer options get bloom-hover treatment, replacing the barely-visible lavender glow
- **D-07:** Glow transitions stay within 150-300ms per UI/UX Pro Max duration-timing guideline
- **D-08:** `prefers-reduced-motion` must be respected — bloom effects use CSS transitions that naturally degrade

### Border & Edge Treatment
- **D-09:** Directional neon accent borders on key interactive cards (category cards, quiz question card) — not every card
- **D-10:** Category cards use mastery color (red/yellow/green) as left border accent — functional AND decorative
- **D-11:** Quiz question card gets neon-cyan left border accent
- **D-12:** Non-interactive cards (stats, history) rely on ghost-border only, no directional accents

### Background Atmosphere
- **D-13:** Add subtle radial gradient to app layout background (dark purple to darker purple)
- **D-14:** One fixed blur blob in top-right corner of app layout — every page inherits it
- **D-15:** Same blob across all pages (dashboard, quiz, summary) — positioned in the app layout, not per-page
- **D-16:** Blob should be subtle enough to not distract during quiz focus

### Claude's Discretion
- Exact blur blob size, opacity, and color
- Exact gradient values for the background
- Whether to keep rounded-xl or reduce border radius on some elements for sharper feel
- Specific bloom-hover shadow values (current class exists, may need tuning)
- Whether quiz-in-progress should dim the blob further for focus

</decisions>

<specifics>
## Specific Ideas

- The landing page's hero panel (`border border-neon-pink/20 bg-card/80 backdrop-blur-2xl rounded-xl`) is the target aesthetic for app cards
- Quiz stats card already uses `glass ghost-border` — this is the pattern to replicate across all cards
- The style reconciliation should feel like walking from the landing page into the app without a visual "drop" — same design language, just applied functionally

</specifics>
