  <action>
    Create the quiz UI component tree. All components are "use client" except the practice page which is a server component that fetches questions then renders client components.

    **src/hooks/use-quiz.ts** -- Orchestration hook:
    - Wraps useQuizStore selectors and actions
    - `startQuiz(mode, questionCount?)`: calls server action to fetch questions (selectPracticeQuestions or selectSimQuestions), then store.initSession()
    - `handleSubmit()`: calls store.submitAnswer(). For practice mode: if correct, triggers confetti + correct sound, sets auto-advance timeout (AUTO_ADVANCE_DELAY_MS). If wrong, triggers wrong sound. For sim mode: just advances.
    - `handleNext()`: calls store.nextQuestion()
    - `handleQuit()`: calls saveQuizResults server action with current answers (partial), then store.quitQuiz(), then router.push("/dashboard")
    - `handleComplete()`: calls saveQuizResults with all answers (complete), returns sessionId for summary page
    - Exposes: currentQuestion, answerState, selectedAnswer, score, streak, progress, isComplete, isLastQuestion, all action functions

    **src/server/actions/quiz.ts** -- Server actions:
    - `fetchQuestions(mode: QuizMode, count?: number)`: creates server Supabase client, calls selectPracticeQuestions or selectSimQuestions, returns PreparedQuestion[]
    - `saveQuizResults(input: SaveQuizResultsInput)`: validates with Zod schema, creates server Supabase client, inserts quiz_session row, inserts quiz_answers rows, returns { sessionId, score, total, passed }
    Per Research code example. Use saveQuizResults for both complete and partial (quit) saves.

    **src/components/quiz/quiz-shell.tsx** -- Main quiz container:
    - Props: { mode: QuizMode, questionCount?: number }

        const ctx = new AudioContext()

        osc.type = type; osc.frequency.value = frequency
        osc.connect(ctx.destination)
        osc.start(); osc.stop(ctx.currentTime + duration / 1000)
