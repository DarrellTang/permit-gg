Fix two independent failure paths that cause the mastery delta banner to silently return null.

**Root cause analysis (from debug/mastery-delta-banner-missing.md):**
1. For anonymous users: `fetchUserAnalytics()` returns null, so pre-drill mastery is never written to sessionStorage, AND post-drill fetch also returns null
2. For first-drill authenticated users: category not found in analytics response (no prior data), so sessionStorage never written
3. Race condition: post-drill fetch may return stale data if saveQuizResults hasn't committed yet

**Fix approach -- two files need changes:**

### File 1: src/components/quiz/quiz-shell.tsx (handleStart function, lines 79-96)

Change the pre-drill mastery snapshot logic to handle both failure modes:

```typescript
if (isDrill && categorySlug) {
  try {
    const analyticsData = await fetchUserAnalytics()
    if (analyticsData) {
      const catMastery = analyticsData.categories.find(
        (c) => c.slug === categorySlug
      )
      // Default to 0 if category has no prior data (first drill)
      sessionStorage.setItem(
        `drill_pre_mastery_${categorySlug}`,
        String(catMastery?.masteryPct ?? 0)
      )
    } else {
      // Anon user or fetch failed -- default to 0% pre-mastery
      sessionStorage.setItem(
        `drill_pre_mastery_${categorySlug}`,
        "0"
      )
    }
  } catch {
    // Analytics fetch failed -- default to 0% pre-mastery
    sessionStorage.setItem(
      `drill_pre_mastery_${categorySlug}`,
      "0"
    )
  }
}
```

Key change: sessionStorage is ALWAYS written when isDrill is true. If analytics returns null or category not found, default to 0%.

3. In the useEffect, after fetching post-drill analytics, add a fallback path:
```typescript
useEffect(() => {
  const preMastery = sessionStorage.getItem(`drill_pre_mastery_${categorySlug}`)
  if (!preMastery) {
    setLoading(false)
    return
  }

  const preValue = Number(preMastery)
  setBefore(preValue)
  sessionStorage.removeItem(`drill_pre_mastery_${categorySlug}`)

  fetchUserAnalytics()
    .then((data) => {
      if (data) {
        const catMastery = data.categories.find((c) => c.slug === categorySlug)
        if (catMastery) {
          setAfter(catMastery.masteryPct)
          return
        }
      }
      // Server fetch returned null or category not found -- compute locally
      computeLocalMastery()
    })
    .catch(() => {
      // Fetch failed -- compute locally
      computeLocalMastery()
    })
    .finally(() => setLoading(false))
}, [categorySlug])
```

4. Add the local computation function inside the component:
```typescript
const computeLocalMastery = () => {
  if (!localAnswers || localAnswers.length === 0) {
    // No local data available -- show drill results as the "after"
    // This shouldn't happen since we're on the summary page with store data
    setAfter(0)
    return
  }
  const categoryAnswers = localAnswers.filter(
    (a) => a.categorySlug === categorySlug
  )
  if (categoryAnswers.length === 0) {
    setAfter(0)
    return
  }
  const correct = categoryAnswers.filter((a) => a.isCorrect).length
  const pct = Math.round((correct / categoryAnswers.length) * 100)
  setAfter(pct)
}
```
