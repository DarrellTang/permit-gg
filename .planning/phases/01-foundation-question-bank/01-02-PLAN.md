  <name>Task 2: Generate ~300 verified CA DMV seed questions and validation script</name>
  <files>supabase/seed.sql, scripts/validate-seeds.ts, tests/seed-questions.test.ts</files>
  <action>
    1. Create `supabase/seed.sql` containing ~300 CA DMV permit test questions as SQL INSERT statements into the seed_questions table. Generate questions from knowledge of the 2026 CA DMV Driver Handbook.

    **Question sourcing strategy (per user decision -- generate from handbook knowledge):**
    - Write original question text (do NOT copy from third-party sites)
    - Each question MUST have: question_text, correct_answer, exactly 3 wrong_answers (plausible distractors), explanation (why the correct answer is right), handbook_reference (specific handbook section/chapter)
    - All 4 answer options must be plausible -- no obviously joke answers
    - Cross-validate facts against current CA law (2026 handbook)

    **Category distribution (minimum 25 per category, weighted by real test emphasis):**
    - Road Signs: ~45 questions (signs, signals, pavement markings)
    - Right-of-Way: ~45 questions (intersections, crosswalks, emergency vehicles)
    - Traffic Laws: ~45 questions (lanes, turns, signals, cell phone laws)
    - Speed Limits: ~30 questions (zones, conditions, school zones)
    - DUI/Drug Laws: ~30 questions (BAC limits for under 21 = 0.01%, penalties)
    - Safe Driving: ~45 questions (following distance, weather, night, emergencies)
    - Parking: ~30 questions (curb colors, restricted zones, parallel parking)
    - Sharing the Road: ~30 questions (pedestrians, cyclists, motorcycles, trucks)

    **Format for each INSERT:**
    ```sql
    INSERT INTO seed_questions (category_id, question_text, correct_answer, wrong_answers, explanation, handbook_reference)
    VALUES (
      (SELECT id FROM categories WHERE slug = 'category-slug'),
      'Question text here?',
      'Correct answer',
      ARRAY['Wrong 1', 'Wrong 2', 'Wrong 3'],
      'Explanation of why the correct answer is right.',
      'CA DMV Handbook, Chapter X: Section Name'
    );
    ```

    2. Create `scripts/validate-seeds.ts` -- A TypeScript script (runnable with `npx tsx`) that:
       - Reads supabase/seed.sql
       - Parses INSERT statements using regex
       - Validates each question has: non-empty question_text (min 10 chars), non-empty correct_answer, exactly 3 non-empty wrong_answers, non-empty explanation (min 10 chars), non-empty handbook_reference
       - Validates each question references a valid category slug
       - Counts questions per category and total
       - Prints a distribution summary table
       - Exits with code 1 if any validation fails, 0 if all pass
       - Reports: total count, per-category count, any categories below 25 minimum

    3. Create/update `tests/seed-questions.test.ts` -- Tests that validate the seed.sql file offline (no Supabase connection needed):
       - Test: "seed.sql contains at least 280 INSERT statements" (allow some margin below 300 target)
       - Test: "all 8 categories are referenced in seed.sql"
       - Test: "each question has exactly 3 wrong answers" (check ARRAY syntax)
       - Test: "no duplicate question_text values"

    4. Run `npx tsx scripts/validate-seeds.ts` to verify seed quality.
    5. Run `npx vitest run tests/seed-questions.test.ts` to verify tests pass.

    IMPORTANT: These are CA DMV-specific facts that MUST be correct:
    - Under 21 BAC limit = 0.01% (zero tolerance)
    - 21+ BAC limit = 0.08%
    - School zone speed = 25 mph when children present
    - Residential speed = 25 mph
    - Business district speed = 25 mph
    - Highway speed = 65 mph (some posted 70 mph)
    - Following distance = 3 seconds minimum
    - Red curb = no parking/stopping, White curb = passenger loading, Green curb = time-limited, Yellow curb = commercial loading, Blue curb = disabled parking
    - Must stop for school bus with flashing red lights
    - Cell phone: hands-free only for 18+, NO phone use at all for under 18
