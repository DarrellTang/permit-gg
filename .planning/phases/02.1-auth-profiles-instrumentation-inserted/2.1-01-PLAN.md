Create two SQL migration files following the existing naming convention (00005_, 00006_).

**00005_create_profiles.sql:**
- CREATE TABLE `profiles` with columns: `id` UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE, `display_name` TEXT, `avatar_url` TEXT, `stripe_customer_id` TEXT (nullable), `tier` TEXT NOT NULL DEFAULT 'free' CHECK (tier IN ('free', 'pro')), `created_at` TIMESTAMPTZ NOT NULL DEFAULT now(), `updated_at` TIMESTAMPTZ NOT NULL DEFAULT now()
- ENABLE ROW LEVEL SECURITY on profiles
- RLS policy: authenticated users SELECT their own profile (auth.uid() = id)
- RLS policy: authenticated users UPDATE their own profile (auth.uid() = id) with WITH CHECK
- CREATE OR REPLACE FUNCTION public.handle_new_user() as TRIGGER LANGUAGE plpgsql SECURITY DEFINER SET search_path = '' that INSERTs into public.profiles (id, display_name, avatar_url) using NEW.id, COALESCE(NEW.raw_user_meta_data ->> 'full_name', NEW.raw_user_meta_data ->> 'name', NEW.email), NEW.raw_user_meta_data ->> 'avatar_url'
- CREATE TRIGGER on_auth_user_created AFTER INSERT ON auth.users FOR EACH ROW EXECUTE FUNCTION public.handle_new_user()
- CREATE OR REPLACE FUNCTION public.update_updated_at() trigger for auto-updating updated_at
- CREATE TRIGGER profiles_updated_at BEFORE UPDATE ON profiles

**00006_user_scope_quiz_tables.sql:**
- ALTER TABLE quiz_sessions ADD COLUMN user_id UUID REFERENCES auth.users(id) (nullable for free quiz)
- ALTER TABLE quiz_answers ADD COLUMN user_id UUID REFERENCES auth.users(id) (nullable for free quiz)
- CREATE INDEX idx_quiz_sessions_user_id ON quiz_sessions(user_id)
- CREATE INDEX idx_quiz_answers_user_id ON quiz_answers(user_id)
- DROP existing open policies: "Allow all access to quiz_sessions" and "Allow all access to quiz_answers"
- New RLS policies for quiz_sessions: authenticated SELECT/INSERT/UPDATE own rows (auth.uid() = user_id)
- New RLS policies for quiz_answers: authenticated SELECT/INSERT own rows (auth.uid() = user_id)
- Anon INSERT policies for both tables: WITH CHECK (user_id IS NULL) — enables the one free quiz for unauthenticated users
- Keep questions table policies unchanged (public read-only)

Use exact SQL patterns from 2.1-RESEARCH.md Pattern 5 and Pattern 6.

**Update src/lib/supabase/client.ts:** Replace `createClient` from `@supabase/supabase-js` with `createBrowserClient` from `@supabase/ssr`. This is required for cookie-based auth to work on the client. The function signature stays the same (`export function createClient()`), just the import and internal call changes. See 2.1-RESEARCH.md Pattern 2.

**Update src/lib/supabase/server.ts:** Wrap the `setAll` callback in a try/catch because Server Components cannot write cookies (only middleware can). The try/catch silently swallows the error in Server Components while allowing it to work in Server Actions and Route Handlers. This is the standard Supabase SSR pattern.

**Create src/middleware.ts:** Following 2.1-RESEARCH.md Pattern 1:
- Import createServerClient from @supabase/ssr
- Create supabase client with cookie getAll/setAll that reads from request.cookies and writes to both request.cookies and response.cookies
- Call `supabase.auth.getClaims()` to refresh auth tokens (use getClaims not getUser for performance — local JWT validation)
- Route protection: if path starts with /dashboard, /practice, /simulated-test, /analytics, or /flashcards AND user is not authenticated (error or no claims), redirect to /login with ?next= return URL parameter
- IMPORTANT: Do NOT protect /login, /, /auth/callback, or any (marketing) routes
- Export matcher config that excludes _next/static, _next/image, favicon.ico, and common image extensions
- NOTE: Free quiz gating will be added to middleware in Plan 02

**Create src/app/auth/callback/route.ts:** Following 2.1-RESEARCH.md Pattern 3:
- Export async GET handler
- Extract `code` and `next` from searchParams
- Sanitize `next` (must start with /, default to /dashboard)
- If code exists, create supabase server client and call exchangeCodeForSession(code)
- On success: redirect to `${origin}${next}` (handle x-forwarded-host for production)
- On error: redirect to /login?error=auth-code-error

**Create src/server/actions/auth.ts:**
- 'use server' directive
- `signOut()`: creates server client, calls supabase.auth.signOut(), redirects to /login
- `signInWithEmail(formData: FormData)`: extracts email/password, calls supabase.auth.signInWithPassword. On error returns { error: string }. On success redirects to /dashboard.
- `signUpWithEmail(formData: FormData)`: extracts email/password/displayName, calls supabase.auth.signUp with data: { display_name }. On error returns { error: string }. On success redirects to /dashboard.
